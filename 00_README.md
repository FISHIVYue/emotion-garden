# 情绪植物树洞｜Codex 开发资料包

## 当前开发结论

第一阶段只做一个 **移动端优先的网页应用（Web App / PWA）**，暂时不接 Apple Watch、Wear OS 或其他手表。

第一版的目标不是做完整商业产品，而是先验证三件事：

1. 用户是否喜欢“情绪被植物表现出来”的核心体验。
2. 用户是否愿意持续进行轻量情绪记录。
3. 精美、安静、有生命感的 UI 是否能构成产品差异。

## 第一版包含

- 欢迎页与首次进入体验
- 植物匹配问卷
- 植物分配结果页
- 今日花园首页
- 情绪记录流程
- 情绪日历
- 匿名树洞/共鸣森林的演示版
- 植物档案与设置
- 本地数据保存
- 移动端响应式布局
- PWA 基础能力

## 第一版不包含

- 智能手表接入
- 医疗诊断
- 真实 AI 情绪识别
- 真实社交系统
- 实时聊天
- 复杂后端
- 支付与订阅
- 多品牌可穿戴设备适配

## 文件说明

- `01_PRODUCT_BRIEF.md`：产品定位与核心原则

- `05_NANO_BANANA_PROMPTS.md`：用于生成素材的提示词模板
- `06_CODEX_BUILD_SPEC.md`：Codex 技术实现要求

- `08_ACCEPTANCE_CHECKLIST.md`：验收清单
- `09_MASTER_PROMPT_FOR_CODEX.md`：可直接粘贴给 Codex 的总任务指令

## 推荐执行顺序

1. 先准备视觉参考图和核心植物素材。
2. 把 `09_MASTER_PROMPT_FOR_CODEX.md` 发给 Codex。
3. 让 Codex 先完成静态页面与导航。
4. 再接入本地数据、问卷结果、植物状态变化。
5. 最后补动画、PWA 与细节优化。
