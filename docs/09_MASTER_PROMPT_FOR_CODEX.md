# 可直接粘贴给 Codex 的总任务

你是一名高级前端工程师和产品实现工程师。请根据当前目录中的产品文档，构建一个名为“情绪花园 / Inner Garden”的移动端优先 Web App。

## 核心目标

这个产品通过植物状态表达用户的日常情绪。它不是养成游戏，不使用生命值、连续打卡或负面惩罚。植物不会因为悲伤、焦虑或疲惫而死亡，而是用露珠、雾、叶片收拢、光线和枝条状态表达情绪。

第一版只做网页/PWA，不接智能手表、不接 AI、不接真实后端。所有数据保存在浏览器本地，但代码结构要为未来的健康数据和后端同步预留接口。

## 实现要求

1. 使用 React + TypeScript 构建。
2. 使用移动端优先响应式设计。
3. 页面包含欢迎、问卷、植物匹配、花园、情绪记录、记录结果、年轮、森林、发布树洞、我的和设置。
4. 使用本地存储保存用户状态、植物状态和情绪记录。
5. 提供合理的模拟树洞数据和规则式植物回应。
6. 加入轻量植物呼吸、雾层移动和页面过渡动效。
7. 支持 `prefers-reduced-motion`。
8. 提供 PWA manifest、图标占位和基础离线支持。
9. 所有视觉颜色、间距、圆角和阴影都通过统一设计 token 管理。
10. 素材缺失时先使用优雅的 CSS/SVG 占位，不要用廉价卡通图片。
11. 不使用付费 API，不要求任何密钥。
12. 不实现真实登录、支付、AI、手表或云数据库。

## 开发步骤

- 初始化项目
- 建立路由

- 完成情绪记录

### 阶段三：年轮与森林

- 完成单日详情

### 阶段四：体验优化

- 增加动效
- 增加 PWA
- 检查响应式
- 检查可访问性
- 完成 README
